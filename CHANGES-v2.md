# API-Imitator v2 🚀

## 版本更新亮点

### 🌟 新增功能

#### 1. 自定义 HTTP 状态码
- 每个 API 配置可以独立设置返回状态码（200, 400, 401, 403, 404, 500 等）
- 支持 100-599 范围内所有状态码
- 参数校验：超出范围会提示错误

#### 2. 自定义响应头
- 支持添加任意自定义响应头
- 场景：`X-Custom-Header`, `Authorization`, `Set-Cookie` 等
- 自动 JSON 序列化/反序列化

#### 3. 延迟响应（网络模拟）
- 支持设置接口响应延迟时间（毫秒）
- 范围：0 - 30000ms（30秒）
- 完美模拟真实网络环境、慢接口场景

#### 4. 配置启用/禁用开关
- 每个 API 配置可单独启用/禁用
- 禁用的配置不会响应请求
- 无需删除配置，快速切换状态

### ⚡ 性能优化

#### 1. Spring Cache 缓存机制
- 所有 API 配置查询结果缓存
- 配置变更时自动清除缓存
- 高并发场景性能提升显著

#### 2. 正则表达式缓存
- 路径匹配的正则表达式编译后缓存
- 避免每次请求重复编译正则
- `ConcurrentHashMap` 线程安全

#### 3. 代码优化
- `ObjectMapper` 使用 Spring 单例注入
- 避免二次正则匹配（matchPath 结果复用）
- 移除不必要的 TransactionTemplate 嵌套查询

### 🛡️ 代码质量提升

#### 1. 全局异常处理器
- 统一错误响应格式
- 参数校验失败友好提示
- 服务器内部错误优雅处理
- 所有异常带时间戳和详细信息

#### 2. 完整日志体系
- SLF4J + Logback 标准日志
- INFO 级别：请求访问日志
- DEBUG 级别：详细调试信息
- WARN/ERROR 级别：异常告警

#### 3. 参数校验增强
- 状态码范围校验（100-599）
- 延迟时间范围校验（0-30000ms）
- 所有校验失败返回友好提示

### 📊 新字段说明

实体类 `ApiConfig` 新增字段：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `statusCode` | Integer | 200 | HTTP 响应状态码 |
| `responseHeaders` | String | null | 响应头（JSON 格式存储） |
| `delayMs` | Integer | 0 | 响应延迟（毫秒） |
| `enabled` | Boolean | true | 配置是否启用 |

### 🔄 配置变更自动缓存清除

缓存自动清除时机：
- ✅ 创建新配置时
- ✅ 更新配置时  
- ✅ 删除配置时

缓存名称：
- `apiConfigs` - API 配置响应 DTO 列表
- `apiConfigEntities` - API 配置实体列表

## 架构设计改进

```
v2 架构改进点：

  ┌─────────────────────────────────────────┐
  │         Spring Cache Abstraction        │  ← 新增缓存层
  └─────────────────────────────────────────┘
                        │
  ┌─────────────────────────────────────────┐
  │        ApiConfigService (带缓存)        │  ← 缓存注解
  └─────────────────────────────────────────┘
                        │
  ┌─────────────────────────────────────────┐
  │    ResponseBuilder (正则表达式缓存)     │  ← 本地缓存
  └─────────────────────────────────────────┘
                        │
  ┌─────────────────────────────────────────┐
  │    DynamicApiController (新功能支持)    │  ← 状态码/响应头/延迟
  └─────────────────────────────────────────┘
```

## 使用示例

### 创建带新功能的 API 配置

```json
POST /api/configs
{
  "path": "/api/users/{id}",
  "method": "GET",
  "description": "获取用户信息（模拟网络延迟）",
  "statusCode": 200,
  "delayMs": 500,
  "enabled": true,
  "responseHeaders": {
    "X-Server": "API-Imitator v2",
    "Cache-Control": "no-cache"
  },
  "fields": [
    {
      "name": "id",
      "type": "INTEGER",
      "value": "{id}"
    },
    {
      "name": "name",
      "type": "STRING",
      "value": "Test User"
    }
  ]
}
```

### 模拟错误场景

```json
{
  "path": "/api/forbidden",
  "method": "GET",
  "statusCode": 403,
  "description": "模拟权限不足"
}
```

### 模拟超时场景

```json
{
  "path": "/api/slow",
  "method": "GET",
  "statusCode": 504,
  "delayMs": 10000,
  "description": "模拟10秒超时"
}
```

## 向后兼容性

✅ **完全兼容 v1 版本配置**
- 旧配置升级后默认 `enabled = true`
- 默认 `statusCode = 200`
- 默认 `delayMs = 0`
- 无需数据迁移，无缝升级
